Pilas (STACK)

¿Para qué sirve una pila?

Una pila es una estructura de datos lineal que funciona bajo el principio LIFO (Last In, First Out), donde el último elemento en ingresar es el primero en salir. Se utiliza en problemas donde es necesario llevar un control secuencial y reversible.

Características de una pila:

  • Acceso restringido a un solo extremo: la cima (TOP).
  • El último dato insertado es el primero en eliminarse.
  • Gestión ordenada y temporal de los datos.
  • Estructura simple pero muy eficiente en memoria.

Funciones principales:

  • push(x): Inserta un elemento en la cima.
  • pop(): Elimina el último elemento insertado.
  • peek(): Obtiene el elemento en la cima sin eliminarlo.
  • isEmpty(): Verifica si la pila está vacía.
  • isFull(): (si es implementada con tamaño fijo).

Colas (QUEUE)

¿Para qué sirve una cola?

Una cola es una estructura de datos lineal que sigue el principio FIFO (First In, First Out), donde el primer elemento en ingresar es el primero en salir. Es ideal para gestionar procesos en espera o flujos ordenados.

Características de una cola:

  • Acceso por dos extremos: frente (front) y final (rear).
  • El primero que entra es el primero que sale.
  • Se usa en sistemas de turnos, impresoras, colas de procesos, etc.
  • Puede ser simple, circular, prioridad o doble cola.

Funciones principales:

  • enqueue(x): Inserta un elemento al final.
  • dequeue(): Elimina el elemento del frente.
  • front(): Muestra el primer elemento sin eliminarlo.
  • isEmpty(): Verifica si la cola está vacía.
  • isFull(): (si la implementación tiene tamaño limitado).

Árboles Binarios (BINARY TREE)

¿Para qué sirve un árbol binario?

Un árbol binario es una estructura de datos no lineal compuesta por nodos, donde cada nodo puede tener como máximo dos hijos (izquierdo y derecho). Se utiliza para representar jerarquías, realizar búsquedas eficientes y organizar datos de forma estructurada.

Características de un árbol binario:

  • Cada nodo tiene como máximo dos hijos.
  • Existe un nodo raíz del cual parten todos los demás.
  • Los nodos sin hijos se llaman hojas.
  • Permite recorridos: inorden, preorden, postorden.

Tipos de árboles binarios:

  • Árbol Binario de Búsqueda (BST): Los valores menores van a la izquierda, los mayores a la derecha.
  • Árbol Binario Completo: Todos los niveles están llenos excepto posiblemente el último.
  • Árbol Binario Balanceado: La altura de los subárboles no difiere en más de 1 (ej: AVL).

Operaciones principales:

  • Insertar: Agrega un nuevo nodo respetando las reglas del árbol.
  • Buscar: Encuentra un nodo con un valor específico.
  • Eliminar: Remueve un nodo manteniendo la estructura.
  • Recorrer: Visita todos los nodos en un orden específico.

Aplicaciones comunes:

  • Sistemas de archivos jerárquicos
  • Bases de datos indexadas
  • Algoritmos de compresión (Huffman)
  • Expresiones aritméticas

Comparación de Números (Mayor de 2 Números)

¿Para qué sirve este ejercicio?

Este proyecto implementa un algoritmo básico de comparación entre dos números para determinar cuál es el mayor. Aunque es un problema simple, es fundamental para entender estructuras de control y toma de decisiones en programación.

Conceptos fundamentales aplicados:

  • Estructuras de control: Uso de condicionales (if-else).
  • Operadores relacionales: Comparación entre valores (>, <, ==).
  • Entrada y salida de datos: Lectura de valores y presentación de resultados.
  • Lógica algorítmica: Base para algoritmos más complejos.

Extensiones posibles del ejercicio:

  • Comparar 3 o más números
  • Ordenar un conjunto de números
  • Encontrar el mayor y menor simultáneamente
  • Implementar con arreglos dinámicos

Aplicación práctica:

Este tipo de lógica se utiliza en sistemas de clasificación, validación de datos, filtros de búsqueda, y como base para algoritmos de ordenamiento más complejos.

Simulación de Estructuras de Datos

¿Para qué sirve esta simulación?

Este proyecto proporciona una representación visual e interactiva de diversas estructuras de datos. Permite comprender de manera práctica cómo funcionan las operaciones sobre estas estructuras mediante simulaciones gráficas.

Características del simulador:

  • Interfaz gráfica: Visualización de operaciones en tiempo real.
  • Interactividad: Permite realizar operaciones paso a paso.
  • Múltiples estructuras: Simula pilas, colas, listas, árboles, etc.
  • Fines educativos: Ideal para el aprendizaje visual de conceptos.

Estructuras que puede simular:

  • Pilas (Push, Pop, Peek)
  • Colas (Enqueue, Dequeue)
  • Listas enlazadas (Insertar, Eliminar, Buscar)
  • Árboles binarios (Recorridos, Inserción)
  • Algoritmos de ordenamiento

Ventajas del aprendizaje visual:

  • Facilita la comprensión de conceptos abstractos
  • Permite experimentar sin escribir código
  • Identifica errores en la lógica de manera visual
  • Refuerza el aprendizaje mediante la práctica interactiva

Tecnologías utilizadas:

El proyecto está desarrollado en Java (98.9%) con componentes visuales CSS (1.1%), utilizando JavaFX o Swing para la interfaz gráfica. Compatible con NetBeans.
